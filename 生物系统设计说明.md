## 一、设计理念

### 1.1 统一生物定义

在循迹之境的战斗系统中，**伙伴（Companion）和怪物（Monster）本质上是同一种存在**，唯一的区别是：

| 类型 | 阵营 | 技能控制 |
|------|------|----------|
| 伙伴 | 友方 | 玩家手动选择技能释放 |
| 怪物 | 敌方 | AI根据意图模式自动选择 |

### 1.2 核心优势

- **单一数据源**：生物属性和技能只定义一次，避免数据不一致
- **易于扩展**：新增生物只需在一个文件中定义，即可作为伙伴或怪物使用
- **命名一致**：同一生物在伙伴和怪物列表中使用相同名称
- **灵活切换**：任何生物都可以在需要时从伙伴变为怪物或反之

## 三、API 使用

### 3.1 导入生物定义

```typescript
// 从 units 导入
import { FlameBearDefinition } from './units';
import { ALL_CREATURES, getCreatureById } from './units';
```

### 3.2 创建伙伴

```typescript
import { createCompanion, FlameBearDefinition } from './units';
import { PlayerUnit } from './battle/PlayerUnit';

// 方式1：使用具体生物定义
const bear: PlayerUnit = createCompanion(FlameBearDefinition);

// 方式2：使用ID查找
const bear2: PlayerUnit = createCompanion(getCreatureById('flame_bear'));
```

### 3.3 创建怪物

```typescript
import { createEnemy, FlameBearDefinition } from './units';
import { EnemyUnit } from './battle/EnemyUnit';

// 同一生物定义，既可以作为伙伴也可以作为怪物
const bearEnemy: EnemyUnit = createEnemy(FlameBearDefinition);
```

### 3.4 获取所有生物

```typescript
import { ALL_CREATURES, getCreaturesByElement } from './units';

// 获取所有生物
const all = ALL_CREATURES; // 20只

// 按属性筛选
const fireCreatures = getCreaturesByElement(ElementType.FIRE);

// 按ID查找
const creature = getCreatureById('flame_bear');
```

## 四、生物定义格式

### 4.1 CreatureDefinition 接口

```typescript
interface CreatureDefinition {
  id: string;           // 唯一标识符
  name: string;        // 显示名称
  description: string;  // 描述文本

  // 属性数值
  level: number;
  maxHp: number;
  attack: number;
  defense: number;
  spAttack: number;
  spDefense: number;
  speed: number;

  // 元素属性（1-2个）
  elements: ElementType[];

  // 技能组（4个）
  skills: Skill[];

  // AI相关（怪物专用）
  aiStrategy?: AIDifficulty;
  intentPattern?: IntentPattern[];
  goldReward?: number;
  expReward?: number;
}
```

### 4.2 定义示例

```typescript
export const FlameBearDefinition: CreatureDefinition = {
  id: 'flame_bear',
  name: '焰爪熊',
  description: '浑身燃烧着烈焰的凶猛巨熊',
  level: 10,
  maxHp: 110,
  attack: 100,
  defense: 85,
  spAttack: 75,
  spDefense: 70,
  speed: 70,
  elements: [ElementType.FIRE],
  skills: createFlameBearSkills(),
  aiStrategy: AIDifficulty.NORMAL,
  intentPattern: [
    { turn: 1, intent: IntentType.ATTACK, power: 60 },
    { turn: 2, intent: IntentType.ATTACK, power: 75 },
    { turn: 3, intent: IntentType.BUFF, probability: 1.0 },
    { turn: 4, intent: IntentType.ATTACK, power: 100 },
  ],
  goldReward: 15,
  expReward: 25
};
```

## 五、技能系统

### 5.1 技能工厂函数

```typescript
// 攻击技能
createCreatureAttackSkill(id, name, power, energyCost, element)

// 物理攻击技能
createCreaturePhysicalSkill(id, name, power, energyCost, element)

// 防御/护盾技能
createCreatureShieldSkill(id, name, shieldAmount)

// 强化技能
createCreatureBuffSkill(id, name, description)

// 治疗技能
createCreatureHealSkill(id, name, healAmount)
```

### 5.2 能量消耗规则

| 威力范围 | 能量消耗 |
|----------|----------|
| 45以下 | 1能量 |
| 46-80 | 2能量 |
| 81-100 | 3能量 |
| 100以上 | 4能量 |

### 5.3 技能组定义

```typescript
function createFlameBearSkills(): Skill[] {
  return [
    createCreatureAttackSkill('flame_bear_inferno', '烈焰冲击', 75, 1, ElementType.FIRE),
    createCreatureShieldSkill('flame_bear_shield', '火焰护盾', 50),
    createCreatureBuffSkill('flame_bear_burn', '燃尽', '3回合后造成50点伤害'),
    createCreatureAttackSkill('flame_bear_explosion', '爆裂拳', 100, 3, ElementType.FIRE),
  ];
}
```

## 六、意图系统（怪物AI）

### 6.1 IntentPattern 接口

```typescript
interface IntentPattern {
  turn: number;           // 回合数
  intent: IntentType;     // 意图类型
  power?: number;          // 威力（攻击意图）
  element?: ElementType;  // 元素（攻击意图）
  target?: 'single' | 'all' | 'self' | 'ally';
  probability?: number;    // 触发概率
}
```

### 6.2 意图类型

```typescript
enum IntentType {
  ATTACK = 1,       // 攻击
  ATTACK_ALL = 2,   // 全体攻击
  BUFF = 3,         // 强化
  DEBUFF = 4,       // 弱化
  HEAL = 5,         // 治疗
  DEFEND = 6,       // 防御
  SPECIAL = 7       // 特殊
}
```

### 6.3 AI策略

```typescript
enum AIDifficulty {
  EASY = 'easy',     // 简单：低优先级行动
  NORMAL = 'normal', // 普通：标准策略
  HARD = 'hard',    // 困难：高优先级行动
  ELITE = 'elite',  // 精英：最优策略
  BOSS = 'boss'     // BOSS：多阶段决策
}
```

## 八、扩展指南

### 8.1 新增生物步骤

1. 在 `src/units/index.ts` 中添加生物定义
2. 定义4个技能（使用技能工厂）
3. 配置AI意图模式（如果是怪物）
4. 添加到 `ALL_CREATURES` 数组

### 8.2 代码示例

```typescript
// 1. 定义技能
function createMyCreatureSkills(): Skill[] {
  return [
    createCreatureAttackSkill('my_atk1', '技能1', 70, 2, ElementType.FIRE),
    createCreatureShieldSkill('my_def1', '技能2', 50),
    createCreatureBuffSkill('my_buf1', '技能3', '强化描述'),
    createCreatureAttackSkill('my_atk2', '技能4', 100, 3, ElementType.FIRE),
  ];
}

// 2. 定义生物
export const MyCreatureDefinition: CreatureDefinition = {
  id: 'my_creature',
  name: '我的生物',
  description: '描述...',
  level: 10,
  maxHp: 100,
  attack: 90,
  defense: 80,
  spAttack: 85,
  spDefense: 75,
  speed: 70,
  elements: [ElementType.FIRE],
  skills: createMyCreatureSkills(),
  aiStrategy: AIDifficulty.NORMAL,
  intentPattern: [...],
  goldReward: 15,
  expReward: 20
};

// 3. 添加到导出列表
export const ALL_CREATURES = [
  // ... 其他生物
  MyCreatureDefinition,
];
```
